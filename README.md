# Downloader Bot 🎬🎵

Телеграм-бот для скачивания:

- **TikTok** — кидаешь ссылку (или список через пробел / с новой строки),
  получаешь видео **без водяного знака**. Качает через глобальный источник
  (tikwm.com API + резерв yt-dlp) — регион не важен, работает и на
  модифицированном/«взломанном» тиктоке.
- **YouTube** — кидаешь ссылку, бот спрашивает **🎬 Видео или 🎵 Аудио (mp3)**
  и присылает файл (ffmpeg подтягивается автоматически, ставить не нужно).
- **Яндекс.Музыка** — трек / альбом / плейлист → присылает **mp3**
  (нужен OAuth-токен, см. ниже).

Каждый файл сначала показывается как анимированная «рекламная» заглушка
(«✨ Здесь могла быть ваша реклама ✨»), которая затем **превращается**
в сам файл.

## 🐳 Запуск в Docker (рекомендуется)

Самый простой путь — «поставил токен и запустил»:

```bash
# 1) создай конфиг из примера и впиши токен от @BotFather
cp data/.env.example data/.env
#   отредактируй data/.env -> BOT_TOKEN=...

# 2) собери и запусти
docker compose up -d --build
```

Готово, бот работает. Логи: `docker compose logs -f`. Остановить: `docker compose down`.

**Про env в Docker (важный момент):** токен можно положить в корневой
`./.env` (он прокидывается через `env_file`) **или** в `./data/.env` на
томе — сработает любой. Бот сам дописывает токены в `./data/.env` при
`/auth`, и этот файл **авторитетнее** корневого: его значения перекрывают
корневой `.env`. Поэтому обновления через `/auth` (Яндекс, куки YouTube)
всегда применяются после рестарта, и держать один и тот же ключ в двух
местах не страшно. Пересобирать после `/auth` ничего не нужно.

ffmpeg ставится в образ через apt — конвертация, склейка и нарезка по
таймкодам работают из коробки.

## Установка вручную (без Docker)

```bash
pip install -r requirements.txt
```

## Токен

Получи токен у [@BotFather](https://t.me/BotFather) и вставь его в файл
`token.txt` (одной строкой), либо задай переменную окружения `BOT_TOKEN`.

```powershell
# вариант с файлом:
"1234567890:AA...ВАШ_ТОКЕН" | Out-File -Encoding ascii token.txt
```

## Токен Яндекс.Музыки (только для Яндекса)

TikTok и YouTube работают без этого. Для Яндекс.Музыки нужен OAuth-токен —
но **готовить его заранее не обязательно**: просто кинь боту ссылку на
Яндекс.Музыку, и он (если ты админ) сам предложит кнопку **«🎵 Войти в
Яндекс»** — войдёшь по коду, и он *сразу докачает* твои ссылки. Токен
сохранится в `.env` автоматически.

Те же входы вручную, если хочется:
- команда **`/auth`** в боте → «Обновить токен Яндекса»;
- скрипт `python get_yandex_token.py` → покажет ссылку и код → полученный
  `access_token` вставить в `.env` как `YANDEX_TOKEN=...`.

Готово. Пиши боту в личку ссылку на TikTok / YouTube / Яндекс.Музыку.

## Прокси для Telegram на VPS

Если VPS не может подключиться к `api.telegram.org`, укажи HTTP- или
SOCKS5-прокси в `.env`:

```env
TELEGRAM_PROXY=socks5://127.0.0.1:10808
```

Прокси применяется и к обычным Bot API-запросам, и к long polling. При
`network_mode: host` контейнер видит локальный порт хоста по адресу
`127.0.0.1`. Сам прокси должен выходить в интернет через внешний маршрут,
на котором Telegram доступен; SOCKS-вход с прямым выходом через тот же VPS
блокировку не обходит.

## Работа в группах и топиках

Бота можно добавить в группу. В форуме (группа с топиками) он отвечает
**в том же топике**, где кинули ссылку. Если ссылок в сообщении нет — в
группе молчит.

⚠️ **Важно:** чтобы бот видел обычные сообщения со ссылками в группе,
выключи ему приватность у [@BotFather](https://t.me/BotFather):
`/setprivacy` → выбрать бота → **Disable**. Иначе Telegram отдаёт боту
только команды и ответы на него.

## Команда админа

- **`/noAds`** — вкл/выкл рекламную заглушку «Здесь могла быть ваша реклама».
  Доступна только админу (id зашит в `ADMIN_ID` в [bot.py](bot.py)).
  Когда выключено — вместо рекламы показывается **случайная гифка загрузки**
  из папки `loading_gifs/`, которая так же превращается в скачанный файл.
  Хочешь свои гифки — просто положи `.gif` в `loading_gifs/`.

- **`/auth`** — обновить авторизацию прямо из бота (только админ):
  - **🎵 Яндекс** — вход по коду (device-flow): бот пришлёт ссылку и код,
    подтверждаешь аккаунт → токен сам сохранится в `.env` и применится
    без перезапуска.
  - **🍪 YouTube** — пришлёшь боту файл `cookies.txt` (экспорт из браузера
    расширением «Get cookies.txt LOCALLY»), он сохранит его и будет
    использовать при 403 с датацентровых IP. Отмена — `/cancel`.

## Большие видео с YouTube (до ~2 ГБ)

Обычный Telegram Bot API ограничивает файлы **50 МБ** — это лимит Telegram.
Бот сам понижает качество видео, пока не влезет; если не влезло даже
минимальное — пишет об этом.

Чтобы слать большие файлы, подними **локальный Bot API server**
([tdlib/telegram-bot-api](https://github.com/tdlib/telegram-bot-api)):

1. получи `api_id` / `api_hash` на [my.telegram.org](https://my.telegram.org);
2. запусти сервер: `telegram-bot-api --api-id=ID --api-hash=HASH --local`;
3. в `.env` бота добавь:
   ```
   LOCAL_BOT_API=http://localhost:8081
   ```

Тогда лимит поднимается до ~1900 МБ и бот качает видео в высоком качестве.
Без сервера всё работает, но в рамках 50 МБ.

Потолок разрешения — **1080p по умолчанию** (бережёт VPS: склейка это
дешёвый ремукс, но 4K это лишний трафик/диск). Нужен 4K — добавь в `.env`:
```
MAX_VIDEO_HEIGHT=2160
```

Если ролик всё равно больше 50 МБ, бот пишет «😏 Губа не дура (лимит 50 МБ)»
и — **если у ролика есть главы/таймкоды** — предлагает кнопки скачать его
**кусками по главам**. Каждый кусок обычно влезает в лимит.
